# Backlinks & Link Outreach — Design Spec

**Date:** 2026-04-07
**Status:** Approved

---

## Overview

Add a `backlinks` Django app to the SEO Bot that provides:
1. **Backlink Monitoring** — fetch and track all inbound links to each store via DataForSEO
2. **Link Outreach** — find prospect websites and send AI-drafted cold emails to request backlinks

---

## Architecture

Single new Django app: `backlinks`  
Follows the same pattern as the existing `keywords` app: models, serializers, views, tasks, DataForSEO client.  
Outreach (prospects + campaigns) lives inside the same app since campaign data is tightly coupled to prospect data.

---

## Data Models

### `Backlink`
Represents a single inbound link pointing to a store's domain.

| Field | Type | Notes |
|-------|------|-------|
| `store` | FK → Store | |
| `source_url` | URLField(2000) | The page linking to you |
| `target_url` | URLField(2000) | The page on your store being linked to |
| `domain_rank` | IntegerField | DR score of linking domain (DataForSEO) |
| `page_rank` | IntegerField | PR score of linking page |
| `anchor_text` | CharField(500) | Link anchor text |
| `is_dofollow` | BooleanField | True = dofollow, False = nofollow |
| `first_seen` | DateField | When DataForSEO first detected this link |
| `last_seen` | DateField | Last time DataForSEO confirmed this link |
| `is_lost` | BooleanField | True when link no longer detected |
| `created_at` | DateTimeField | auto |
| `updated_at` | DateTimeField | auto |

**Meta:** `unique_together = [("store", "source_url", "target_url")]`

---

### `BacklinkSnapshot`
Daily aggregate counts per store — used for charts.

| Field | Type | Notes |
|-------|------|-------|
| `store` | FK → Store | |
| `date` | DateField | |
| `total_count` | IntegerField | All backlinks |
| `dofollow_count` | IntegerField | Dofollow only |
| `lost_count` | IntegerField | Lost since previous snapshot |
| `new_count` | IntegerField | New since previous snapshot |

**Meta:** `unique_together = [("store", "date")]`

---

### `OutreachProspect`
A website targeted for a backlink request.

| Field | Type | Notes |
|-------|------|-------|
| `store` | FK → Store | |
| `website_url` | URLField(2000) | Target website |
| `contact_email` | CharField(255) | Email to send outreach to |
| `domain_rank` | IntegerField | DR score from DataForSEO |
| `niche_relevance_score` | IntegerField | 0-100, AI-generated by Claude |
| `source` | CharField | choices: `manual`, `auto_suggested` |
| `status` | CharField | choices: `new`, `emailed`, `followed_up`, `replied`, `won`, `rejected` |
| `notes` | TextField | Optional manual notes |
| `created_at` | DateTimeField | auto |
| `updated_at` | DateTimeField | auto |

---

### `OutreachCampaign`
A single outreach email sent to a prospect.

| Field | Type | Notes |
|-------|------|-------|
| `prospect` | FK → OutreachProspect | |

| `smtp_port` | IntegerField | e.g. 587 |
| `smtp_username` | CharField(255) | |
| `smtp_password` | EncryptedCharField | |
| `smtp_from_email` | CharField(255) | |
| `preferred_method` | CharField | choices: `gmail`, `smtp` |

## API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/backlinks/` | GET | List backlinks for a store (query param: `?store_id=`) |
| `/api/v1/backlinks/refresh/` | POST | Trigger DataForSEO fetch (Celery task) |
| `/api/v1/backlinks/snapshot/` | GET | Daily history for chart (query param: `?store_id=`) |
| `/api/v1/backlinks/summary/` | GET | Totals: count, dofollow %, new, lost |
| `/api/v1/backlinks/prospects/` | GET, POST | List prospects / add manually |
| `/api/v1/backlinks/prospects/suggest/` | POST | Auto-suggest via DataForSEO |
| `/api/v1/backlinks/prospects/{id}/email/` | POST | AI-draft + send outreach email |
| `/api/v1/backlinks/prospects/{id}/status/` | PATCH | Update prospect status |
| `/api/v1/backlinks/campaigns/` | GET | List all sent emails |
| `/api/v1/backlinks/email-config/` | GET, POST, PUT | Manage Gmail/SMTP settings |

## DataForSEO Integration

**`BacklinksClient`** — new client at `backlinks/dataforseo_client.py`, same pattern as `keywords/dataforseo_client.py`.

### Backlink Monitoring
- Endpoint: `backlinks/summary/live`
- Input: store domain
- Output: list of backlinks with source URL, target URL, anchor text, dofollow flag, domain rank, page rank, first/last seen
- Logic: compare against existing DB records, mark `is_lost=True` for missing ones, create new records for new ones, save daily `BacklinkSnapshot`

### Prospect Suggestions
- Endpoint: `backlinks/competitors/live`
- Input: store domain + top keywords
- Output: domains linking to competitors but not to this store, ranked by domain rank
- Claude then scores each for niche relevance (0-100) using store niche context

---

## AI Integration (Claude)

### Prospect Relevance Scoring
- Input: prospect website URL, store name, store niche/keywords
- Claude reads prospect page content, scores relevance 0-100
- Stored in `OutreachProspect.niche_relevance_score`

### Outreach Email Generation
- Input: prospect website URL, store name, store niche, anchor text goal, contact name (if known)
- Claude reads the prospect's page, identifies what they write about, writes a personalized cold email
- Tone: professional, concise, value-first (not spammy)
- Output: subject line + email body
- Email is sent immediately (no review step)

---

## Email Sending

### Gmail (OAuth2)
- Store `gmail_refresh_token` encrypted using existing `EncryptedCharField`
- Use Google Gmail API (`google-auth`, `google-api-python-client`)
- OAuth2 flow: frontend redirects to Google consent screen, callback stores refresh token

### SMTP
- Credentials stored encrypted in `EmailConfig`
- Python `smtplib` with STARTTLS
- Supports any provider (Outlook, custom domain, etc.)

## Frontend

New **Backlinks** page added to the sidebar nav (between Keywords and Technical Audit).

### Tab 1 — Overview
- Summary cards: Total Backlinks, Dofollow %, New This Week, Lost This Week
- Recharts line chart: backlink count over time (uses `BacklinkSnapshot`)
- Table: source domain, anchor text, DR score, dofollow badge, first seen date, lost badge

### Tab 2 — Prospects
- "Auto-suggest" button → triggers suggest task, shows loading state
- "Add manually" button → inline form (URL + contact email)
- Table: website URL, DR score, relevance score, status badge, "Send Email" button per row
- "Send Email" calls AI draft + send endpoint, shows spinner while sending

### Tab 3 — Campaigns
- Table: prospect, subject, sent date, via badge (Gmail/SMTP), reply received badge
- Click row to expand full email body

### Settings page
- New "Email / Outreach" section:
  - Gmail: "Connect Gmail" OAuth button + connected account display
  - SMTP: form for host, port, username, password, from email

## Error Handling

- DataForSEO errors: log and return 502 with message, don't crash task
- Email send failures: log error, mark campaign with `sent_at=None`, surface in UI
- Gmail token expiry: auto-refresh using stored refresh token; if refresh fails, show reconnect prompt in UI
- Claude API errors: log and return 503, prospect status unchanged
